# Mouse Wheel Running — Circadian Activity Analysis

Analyzes voluntary wheel-running data from multi-channel recording systems to characterize circadian rhythms, activity profiles, and behavioral patterns across individual mice.

---

## Repository Contents
| File | Description |
|------|-------------|
| `combined_channels.xlsx` | Raw wheel-running data across 10 channels, plus generated pivot table sheets |
| `pv_table.ipynb` | Main analysis notebook — data cleaning, pivot tables, modeling, and visualization |
| `agg.py` | Preprocessing script (reference only — raw per-day files not included) that merged raw Excel exports into `combined_channels.xlsx` |

## Data

**`combined_channels.xlsx`** contains 12 sheets:

- **`Channel_1` through `Channel_10`** — All 10 channels were recorded simultaneously at 5-minute intervals and originally saved together in a single file. This workbook is that combined data reorganized into per-channel sheets for easier analysis. Columns: `Int`, `Ch`, `Date`, `Time`, `Wheel (counts)`, `Wheel Accum (counts)`
- **`pv_daily`** — Daily total wheel counts per channel (generated by notebook)
- **`pv_hourly`** — Hourly total wheel counts per channel (generated by notebook)

---

## Notebook Overview (`pv_table.ipynb`)

### 1. Data Loading & Cleaning
- Reads all channel sheets from the Excel file and concatenates into a single DataFrame
- Drops empty trailing rows, casts channel numbers back to `int`, and parses `Date`/`Time` into a combined `datetime` column

### 2. Pivot Tables
- **Daily pivot** — total wheel counts per channel per calendar date
- **Hourly pivot** — total wheel counts per channel per hour of day (0 = midnight, 23 = 11 PM)
- Both tables are saved back to `combined_channels.xlsx` as new sheets

### 3. Visualization
- Line graphs of daily and hourly activity, with each channel as a separate series

### 4. Cosinor Model
- Fits a single-component cosine curve to each channel's mean hourly activity profile
- Extracts three circadian parameters per mouse:
  - **MESOR** — mean activity level (midline of the wave)
  - **Amplitude** — strength of the day/night contrast
  - **Phase** — hour of peak activity
- Reports R² goodness-of-fit and plots observed vs. fitted curves per channel

### 5. Cyclic Logistic Regression
- Models the probability of a mouse being active (running at all) at each hour of day
- Hour is cyclically encoded using sine/cosine features so that 11 PM and midnight are treated as adjacent
- Reports balanced accuracy per channel and plots hourly probability curves

### 6. PCA — Behavioral Profiling
- Builds a multi-feature behavioral fingerprint for each mouse:
  - Total running volume
  - Dark/light activity ratio (nocturnality index)
  - Day-to-day coefficient of variation (consistency)
  - Peak activity hour (from cosinor)
  - Mean bout length and number of bouts
- Standardizes features and reduces to 2 principal components for visualization
- Produces a scatter plot of mouse behavioral profiles and a loadings biplot

### 7. Artifact Removal & Model Re-Run
- Identifies and removes zero-activity days (likely equipment/husbandry artifacts)
- Re-runs cosinor, logistic regression, and PCA on the cleaned dataset
- Compares parameter estimates and model accuracy before and after cleaning

### 8. Actograms
- Plots a double-plotted actogram for each mouse showing hour-by-hour activity across all study days
- Dark phase shading based on vivarium light schedule (lights off at 19:00, lights on at 07:00)

### 9. Correlation Heatmap
- Computes pairwise Pearson correlations of daily activity totals across all 10 channels
- Visualized as an annotated heatmap to identify shared environmental influences or individual differences

## Usage

1. Place `combined_channels.xlsx` in the same directory as `pv_table.ipynb`
2. Open the notebook and run all cells in order
3. Pivot table sheets (`pv_daily`, `pv_hourly`) will be written back to the Excel file
4. All plots render inline; no output files are saved by default

> **Note:** The artifact-removal section splits data at `2022-01-14`. Adjust this cutoff date and the vivarium light schedule (`lights_off=19`, `lights_on=7`) to match your experimental setup before re-running models.
